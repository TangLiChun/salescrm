    高级：首次初始化设置

    以下仅在数据库首次初始化时使用，正常运行后请勿随意修改。

    AI 与搜索

    AI 线索发现依赖 LLM；搜索引擎 Key 为可选增强，全部留空时默认使用 DuckDuckGo。

    LLM（AI 线索发现必填）
    智谱联网搜索（推荐）

    使用智谱 Web Search API。若 LLM Base URL 为 open.bigmodel.cn，可留空 API Key 并复用上方 LLM Key。

    Bright Data Google SERP

    Bright Data SERP API，自动兼容 JSON / Markdown / HTML 响应。Zone 默认输出为 Markdown 时可保持「自动」。

    Bright Data 社交媒体 Profiles

    按 URL 抓取 LinkedIn / X / Facebook profile（Bright Data Scraper API）。与 SERP 共用 API Key；discover_leads 会从搜索结果自动提取 URL 并补充抓取。

    Shodan

    互联网资产搜索，discover_leads 会按 PeeringDB 关键词用 org filter 补充 ASN/域名。带 filter 的查询消耗 query credits。

    其他搜索引擎 API（可选）

    线索导入

    控制哪些邮箱可以被导入。白名单非空时仅导入匹配项；黑名单始终排除。

    导入过滤（黑名单 / 白名单）

    每行一条：域名（如 example.com）或带 * 的邮箱模式。

    自动化

    让服务在后台按间隔自动运行 AI 线索发现并导入联系人（按邮箱去重）。
